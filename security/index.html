02

The Three HIPAA Safeguards

Administrative, physical, and technical — all implemented.

Administrative

  • Workforce HIPAA training, documented + dated
  • Role-based access, least privilege by default
  • Sanction policy for violations
  • Annual risk assessment
  • Incident response plan, drilled
  • Vendor BAA management

Physical

  • Azure data centers — SOC 2 / ISO 27001 facilities
  • No PHI on local devices, ever
  • Workstation security policy
  • Encrypted device requirement
  • Media disposal controls
  • US-region data residency

Technical

  • Encryption in transit (TLS 1.3) + at rest (AES-256)
  • Always Encrypted for PHI columns
  • Customer-managed keys in Key Vault (HSM)
  • Unique user IDs + MFA
  • Automatic logoff
  • Audit controls on every PHI access
03

Technical Controls

How PHI is actually protected, end to end.

Encryption everywhere

TLS 1.3 in transit. AES-256 + TDE at rest. PHI columns use Always Encrypted so even DBAs can't read them.

Customer-managed keys

Encryption keys in Azure Key Vault, HSM-backed. EA controls the keys, not the cloud.

Network isolation

No public IPs on the data tier. Private Endpoints for SQL, Storage, Key Vault. VNet + Azure Firewall.

Identity + access

Microsoft Entra ID, MFA enforced, PIM for time-bound elevation. No standing admin access.

Audit on every read

Every PHI access writes an immutable audit log entry. 7-year retention. Queryable for any investigation.

AI minimization

The LLM only sees what a task needs. De-identified data for non-clinical work. PHI stays inside the BAA boundary.

Breach detection

Microsoft Sentinel rules flag anomalous PHI export, after-hours access, and geographic anomalies in real time.

Breach notification

Documented 60-day notification procedure per the HIPAA Breach Notification Rule, ready before launch.
